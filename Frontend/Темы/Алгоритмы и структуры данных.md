# Алгоритмы и структуры данных

[[Frontend Roadmap|← карта]] · база: [[JavaScript]] · подготовка: [[Подготовка к собеседованию]]

> [!info] Зачем фронтендеру
> На вёрстке алгоритмы почти не нужны, но на **собеседованиях** их спрашивают (особенно в крупных компаниях). Цель — не стать олимпиадником, а уметь решать типовые задачи и оценивать сложность кода. Глубоко с уровня [[Frontend · Junior+]].

## 📐 Сложность — Big-O (это must)
- O(1) — константа, O(n) — линейно, O(n²) — вложенные циклы, O(log n) — бинарный поиск.
- Уметь оценить сложность своего кода по времени и памяти.
- Почему вложенный `.find()` внутри `.map()` — это O(n²), и как заменить на Map за O(n).

## 🧱 Структуры данных
- **Массив** — доступ по индексу O(1), поиск O(n).
- **Хеш-таблица** (объект / `Map`) — поиск/вставка O(1). Главный инструмент оптимизации.
- **Множество** (`Set`) — уникальность, быстрая проверка наличия.
- **Стек** (LIFO) и **очередь** (FIFO) — на базе массива.
- **Связный список**, **дерево**, **граф** — понимать идею и обход (DFS/BFS), реализовывать по необходимости.

## 🧠 Приёмы
- Два указателя (two pointers).
- Скользящее окно (sliding window).
- Рекурсия и когда она уместна (обход дерева, например).
- Хеш-таблица для замены вложенного перебора.
- Сортировка (понимать идею, не писать с нуля — есть `.sort()`).

## ⭐ Типовые задачи на собесе (frontend)
- FizzBuzz, разворот строки, проверка на палиндром/анаграмму.
- Найти дубликаты / уникальные элементы массива.
- Сумма двух чисел (Two Sum) — классика на хеш-таблицу.
- Группировка (`groupBy`), «сплющить» вложенный массив (`flatten`).
- Глубокое клонирование объекта (`deepClone`).
- Реализовать `debounce` / `throttle` → есть в [[JavaScript]].
- Реализовать `map`/`filter`/`reduce` своими руками.

## 🏋️ Где тренироваться
- **LeetCode** (Easy → Medium), фильтр по тегам Array / Hash Table / String.
- **Codewars** (kata 7–5 kyu для старта).
- См. также [[Ресурсы — Frontend]].

> [!tip] Стратегия
> Реши 30–50 лёгких задач, чтобы набить руку на массивах/строках/хеш-таблицах. Этого хватает для большинства собеседований на junior–middle. Не уходи в hard — там diminishing returns.
