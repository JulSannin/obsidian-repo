# Frontend · Intern (Стажёр)

[[Frontend Roadmap|← к карте]] · следующий: [[Frontend · Junior]]

> [!abstract] Цель этапа
> Уметь **сверстать статическую страницу по макету** и оживить её базовым JavaScript. Освоить инструменты: редактор, браузер, Git. Срок: ~1–3 месяца.

## ✅ Что освоить

### Веб и инструменты
- Как работает веб: клиент ↔ сервер, что такое браузер, URL, что происходит при вводе адреса (база). → [[Сеть и HTTP]]
- Редактор кода (VS Code): расширения, форматирование, горячие клавиши.
- DevTools браузера: вкладки Elements и Console.
- Терминал: базовые команды (`cd`, `ls`, `mkdir`).

### HTML → [[HTML и доступность]]
- Структура документа, основные теги, семантика (`header`, `main`, `section`, `article`, `footer`).
- Списки, ссылки, изображения, таблицы.
- Формы и инпуты, атрибуты, `label`.

### CSS → [[CSS и вёрстка]]
- Селекторы, специфичность, каскад, наследование.
- Box model, `display`, `position`, единицы (px, %, rem, em).
- **Flexbox** — главный инструмент вёрстки на старте.
- Базовый адаптив через media-queries.

### JavaScript → [[JavaScript]]
- Переменные (`let`/`const`), типы данных, операторы.
- Условия, циклы, функции.
- Массивы и объекты, базовые методы массивов (`map`, `filter`, `forEach`).
- Поиск элементов в DOM и обработка кликов (`querySelector`, `addEventListener`).

### Git → [[Git и инструменты]]
- `init`, `add`, `commit`, `push`, `pull`, `clone`.
- Что такое репозиторий, коммит, ветка.
- Завести аккаунт на GitHub, запушить первый проект.

## 🛠️ Практика
- Свёрстанный лендинг по готовому макету (Figma) — см. [[Frontend-проекты]].
- Pixel perfect — учись попадать в макет.

## 🎓 Критерий перехода на [[Frontend · Junior]]
- [ ] Верстаю адаптивный лендинг по макету без подсказок.
- [ ] Понимаю Flexbox и box model.
- [ ] Пишу простые функции и обработчики событий на JS.
- [ ] Веду проект в Git и публикую на GitHub Pages.
