# Frontend · Junior (Джуниор)

[[Frontend · Intern|← Intern]] · [[Frontend Roadmap|карта]] · следующий: [[Frontend · Junior+]]

> [!abstract] Цель этапа
> Делать **интерактивные интерфейсы на React**, получать данные из API и собирать проект современными инструментами. Срок: ~3–6 месяцев. Это уровень первой работы/стажировки.

## ✅ Что освоить

### Углублённый JavaScript → [[JavaScript]]
- Замыкания, область видимости, `this`, контекст.
- Прототипы и прототипное наследование.
- **Асинхронность**: колбэки → промисы → `async/await`.
- `fetch`, работа с JSON, обработка ошибок.
- **Event loop** — как JS выполняет код (must have для собеса).
- ES-модули (`import`/`export`).

### CSS на уровне продакшена → [[CSS и вёрстка]]
- **Grid** в дополнение к Flexbox.
- Препроцессор (Sass) или Tailwind CSS.
- Методология именования (BEM).
- CSS-переменные, transitions, базовые анимации.

### React → [[React и экосистема]]
- JSX, компоненты, пропсы, композиция.
- Состояние: `useState`, `useEffect`.
- Списки и ключи, условный рендер.
- Формы (controlled inputs).
- Запросы к API внутри компонентов.
- React Router — навигация между страницами.

### Сборка и инструменты → [[Git и инструменты]]
- **npm/pnpm**: `package.json`, скрипты, зависимости.
- **Vite** — создать и собрать проект.
- ESLint + Prettier — линтинг и автоформат.
- Git: ветки, merge, разрешение конфликтов, Pull Request.

### Сеть → [[Сеть и HTTP]]
- HTTP-методы (GET/POST/PUT/DELETE), статус-коды.
- Что такое REST API, как читать документацию API.
- Вкладка Network в DevTools.
- Что такое CORS (на уровне «почему ошибка»).

## 🛠️ Практика
- SPA на React с данными из публичного API (погода, фильмы, todo с бэкендом).
- Деплой на Vercel / Netlify. → [[Frontend-проекты]]

## 🎓 Критерий перехода на [[Frontend · Junior+]]
- [ ] Собираю React-приложение из нескольких страниц с роутингом.
- [ ] Получаю и отображаю данные из REST API, обрабатываю загрузку/ошибки.
- [ ] Понимаю и могу объяснить event loop, промисы, замыкания.
- [ ] Работаю через ветки и Pull Request'ы.
