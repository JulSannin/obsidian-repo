# Backend · Intern (Стажёр)

[[Backend Roadmap|← карта]] · следующий: [[Backend · Junior]]

> [!abstract] Цель этапа
> Понять, **как работает сервер**, поднять простой HTTP-сервер на Node.js, освоить базы данных и SQL на уровне CRUD, уверенно жить в терминале. Срок: ~1–3 месяца (при готовом JS).

## ✅ Что освоить

### Как работает backend → [[Сеть и HTTP]]
- Клиент ↔ сервер ↔ база данных — полный путь запроса.
- Что такое сервер, порт, процесс, переменные окружения.
- HTTP с серверной стороны: запрос, ответ, статус-коды, заголовки.
- Что такое API и зачем он нужен.

### Терминал и ОС → [[Серверная архитектура и DevOps]]
- Linux/Unix базово: навигация, файлы, права, процессы.
- CLI: `cd`, `ls`, `cat`, `grep`, `chmod`, пайпы.
- SSH — что это и зачем.

### Node.js → [[Node.js]]
- Что такое Node.js и чем серверный JS отличается от браузерного (нет DOM, есть `fs`, `process`).
- Запуск скриптов, модули (CommonJS и ESM).
- npm/pnpm, `package.json`, установка пакетов.
- Встроенные модули: `fs`, `path`, `http`.
- Поднять простой HTTP-сервер вручную, затем на **Express**.
- Менеджер версий Node (nvm/fnm), `nodemon` для разработки.

### Базы данных → [[Базы данных и SQL]]
- Реляционные vs нереляционные БД (в общих чертах).
- **SQL**: `SELECT`, `INSERT`, `UPDATE`, `DELETE` (CRUD).
- `WHERE`, `ORDER BY`, `LIMIT`, простые `JOIN`.
- Установить **PostgreSQL**, подключиться, создать таблицу.
- Первичный ключ, типы данных, базовые связи.

### Git → [[Git и инструменты]]
- Повтор/закрепление: ветки, коммиты, Pull Request (то же, что во frontend).

## 🛠️ Практика
- HTTP-сервер на Express, отдающий JSON.
- Консольное приложение, читающее/пишущее в БД (todo в PostgreSQL). → [[Backend-проекты]]

## 🎓 Критерий перехода на [[Backend · Junior]]
- [ ] Поднимаю Express-сервер и отдаю ответ на запрос.
- [ ] Пишу базовые SQL-запросы (CRUD + WHERE + JOIN).
- [ ] Понимаю путь запроса: браузер → сервер → БД → ответ.
- [ ] Уверенно работаю в терминале.
