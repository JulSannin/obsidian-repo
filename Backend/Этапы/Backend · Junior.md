# Backend · Junior (Джуниор)

[[Backend · Intern|← Intern]] · [[Backend Roadmap|карта]] · 🎯 цель трека

> [!abstract] Цель этапа
> Самостоятельно **спроектировать и написать REST API** с базой данных и авторизацией, обработкой ошибок и валидацией, затем задеплоить его. Срок: ~3–6 месяцев. Это уровень первой backend-работы.

## ✅ Что освоить

### REST API → [[REST API на сервере]]
- Проектирование маршрутов (routes), ресурсы и эндпоинты.
- Контроллеры, middleware, обработка запроса/ответа.
- Корректные HTTP-методы и статус-коды.
- **Валидация входных данных** (Zod / Joi) — обязательно.
- Централизованная **обработка ошибок**.
- CORS, парсинг тела, query/params.
- Документация API (Swagger/OpenAPI — базово).

### Базы данных глубже → [[Базы данных и SQL]]
- Сложнее `JOIN`, агрегатные функции, `GROUP BY`.
- Индексы — зачем и когда.
- Связи: один-ко-многим, многие-ко-многим.
- **ORM/Query Builder**: Prisma (рекомендуется для TS) или Knex.
- Миграции схемы.
- Транзакции (база).
- Знакомство с NoSQL (MongoDB) — обзорно.

### Аутентификация → [[Аутентификация и безопасность]]
- Хеширование паролей (**bcrypt/argon2**) — никогда не в открытом виде.
- Регистрация и вход.
- **JWT** vs сессии + cookies.
- Middleware авторизации, защищённые маршруты.
- Роли и доступы (базово).
- OWASP Top-10 на уровне «что это и как не попасть»: SQL-инъекции, XSS, и т.д.

### Node.js / TypeScript → [[Node.js]] · [[TypeScript]]
- Асинхронность на сервере: промисы, `async/await`, потоки (streams) базово.
- Переход проекта на **TypeScript** (типизация роутов, моделей, конфигов).
- Переменные окружения (`dotenv`), конфигурация.
- Логирование (pino/winston).

### Архитектура и деплой → [[Серверная архитектура и DevOps]]
- Структура проекта: разделение на слои (routes / controllers / services / models).
- Паттерн MVC, принцип единственной ответственности.
- **Тестирование API**: Vitest/Jest + Supertest.
- **Docker** — базово (контейнеризировать приложение и БД).
- **Деплой**: Railway / Render / Fly.io, базовый CI (GitHub Actions).

## 🛠️ Практика
- Полноценный REST API: аутентификация, CRUD, связи, валидация, тесты.
- Соедини со своим React-фронтендом → **fullstack-проект**. → [[Backend-проекты]]

## 🎓 Ты — Junior backend, если
- [ ] Пишу REST API с базой данных и авторизацией с нуля.
- [ ] Валидирую вход и централизованно обрабатываю ошибки.
- [ ] Безопасно храню пароли и секреты.
- [ ] Покрываю ключевую логику тестами.
- [ ] Деплою приложение и подключаю к нему фронтенд.

> [!success] Что дальше (Middle backend)
> Кэширование (Redis), очереди сообщений, микросервисы, масштабирование, продвинутый DevOps (CI/CD, Kubernetes), мониторинг и нагрузочное тестирование. Это уже за рамками карты Intern → Junior.
