# Серверная архитектура и DevOps

[[Backend Roadmap|← карта]] · смежное: [[Node.js]] · [[REST API на сервере]]

> [!info] Зачем
> Код, который работает, и код, который можно поддерживать и задеплоить, — разные вещи. Этот раздел про структуру проекта и про то, как довести бэкенд до живого сервера.

## 🟢 Intern — окружение → [[Backend · Intern]]
- Терминал и Linux базово: файлы, права, процессы, пайпы.
- SSH — подключение к удалённому серверу.
- Переменные окружения, `.env` и `.gitignore` (секреты не в Git!).

## 🟡 Junior — архитектура и деплой → [[Backend · Junior]]

### Структура проекта
- Разделение на слои: **routes → controllers → services → models/repositories**.
- Паттерн **MVC**, тонкие контроллеры, логика в сервисах.
- Принцип единственной ответственности, переиспользуемые модули.
- Конфигурация через окружение (dev/prod).
- Логирование (pino/winston) и обработка ошибок как сквозная забота.

### Тестирование → перекликается с [[Тестирование]]
- Unit-тесты сервисов (Vitest/Jest).
- Интеграционные тесты API через **Supertest**.
- Тестовая БД / моки.

### DevOps базово
- **Docker**: контейнеризировать приложение и БД, `Dockerfile`, `docker-compose`.
- **Деплой**: Railway / Render / Fly.io (просто для старта), VPS — обзорно.
- Базовый **CI** (GitHub Actions): прогон тестов и линтера на пуш.
- Мониторинг ошибок (Sentry) — базово.
- Процесс-менеджер для прода (pm2) — обзорно.

## 💡 Пример структуры проекта
```text
src/
├── routes/         # маршруты (только маршрутизация)
│   └── users.ts
├── controllers/    # принять запрос → вызвать сервис → вернуть ответ
│   └── users.ts
├── services/       # бизнес-логика
│   └── users.ts
├── models/         # доступ к данным (Prisma / SQL)
│   └── users.ts
├── middleware/     # auth, валидация, обработка ошибок
├── config/         # переменные окружения, подключение к БД
└── app.ts          # сборка Express-приложения
```

**Минимальный Dockerfile**:
```dockerfile
FROM node:22-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
EXPOSE 3000
CMD ["node", "dist/app.js"]
```
