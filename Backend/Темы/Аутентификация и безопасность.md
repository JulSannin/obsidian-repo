# Аутентификация и безопасность

[[Backend Roadmap|← карта]] · смежное: [[REST API на сервере]] · [[Сеть и HTTP]]

> [!warning] Цена ошибки
> Здесь ошибки стоят дороже всего: утечки паролей, взломы, кража данных. Базовую безопасность джуниор обязан знать. **Главное правило: никогда не храни пароли в открытом виде.**

## 🟡 Junior → [[Backend · Junior]]

### Аутентификация и авторизация
- Разница: **аутентификация** (кто ты) vs **авторизация** (что тебе можно).
- **Хеширование паролей**: bcrypt / argon2 (с солью). Не шифрование, не обратимо.
- Регистрация и вход: проверка пароля через сравнение хешей.
- **JWT** (JSON Web Token): структура, подпись, срок жизни.
- **Сессии + cookies** как альтернатива JWT — когда что выбирать.
- Где хранить токен (httpOnly cookie безопаснее localStorage) → перекликается с [[Сеть и HTTP]].
- Middleware авторизации, защищённые маршруты.
- Роли и права доступа (RBAC, базово).
- Refresh-токены (база).

### Безопасность (OWASP Top-10 для джуна)
- **SQL-инъекции** → использовать параметризованные запросы / ORM.
- **XSS** → экранирование вывода, валидация ввода.
- **CSRF** → токены, SameSite cookies.
- Секреты в переменных окружения, **не в коде и не в Git** (`.gitignore`).
- HTTPS, security-заголовки (helmet в Express).
- Rate limiting — защита от перебора.
- Принцип наименьших привилегий, не отдавать лишнее в ответах (пароли, токены).

## 💡 Примеры кода

**Регистрация: хешируем пароль**:
```ts
import bcrypt from 'bcrypt';

async function register(email: string, password: string) {
  const passwordHash = await bcrypt.hash(password, 10); // соль + хеш
  return prisma.user.create({ data: { email, passwordHash } });
  // В БД хранится ХЕШ, а не пароль
}
```

**Вход: сравниваем и выдаём JWT**:
```ts
import jwt from 'jsonwebtoken';

async function login(email: string, password: string) {
  const user = await prisma.user.findUnique({ where: { email } });
  if (!user) throw new Error('Неверные данные');

  const ok = await bcrypt.compare(password, user.passwordHash);
  if (!ok) throw new Error('Неверные данные'); // не уточняем, что именно неверно

  return jwt.sign({ userId: user.id }, process.env.JWT_SECRET!, { expiresIn: '15m' });
}
```

**Middleware защиты маршрута**:
```ts
function auth(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'Нет токена' });
  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET!);
    next();
  } catch {
    res.status(401).json({ error: 'Невалидный токен' });
  }
}
```
