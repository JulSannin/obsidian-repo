# 🛠️ Backend-проекты

[[Backend Roadmap|← карта]] · общий хаб: [[Карта обучения]]

> [!tip] Главное
> Backend учится через сборку API. Каждый проект → GitHub + README + задеплоенный сервер (живая ссылка на эндпоинты или Swagger).

## 🟢 Intern → [[Backend · Intern]]
Цель: поднять сервер и потрогать БД.
- **HTTP-сервер на Express**, отдающий JSON по нескольким маршрутам.
- **Консольное todo** с хранением в PostgreSQL (CRUD через SQL).
- **Парсер/скрипт** на Node: читает файл, обрабатывает, пишет результат.

## 🟡 Junior → [[Backend · Junior]]
Цель: полноценный REST API.
- **REST API для блога/заметок**: CRUD, связи (пользователь → посты), валидация.
- **API с аутентификацией**: регистрация, вход, JWT, защищённые маршруты → [[Аутентификация и безопасность]].
- **URL-shortener** или **API погоды-прокси** с кэшем.
- **API интернет-магазина**: товары, корзина, заказы, роли (user/admin).
- Обязательно: валидация, обработка ошибок, тесты (Supertest), Docker, деплой.

## 🟣 Fullstack — соединяем треки
> [!success] Самый ценный проект
> Возьми свой фронтенд из [[Frontend-проекты]] и подключи к **собственному** backend. Например: канбан-доска (React) + Node/Express API + PostgreSQL + авторизация. Это показывает, что ты видишь систему целиком — сильнейший сигнал для работодателя.

## 📋 Чек-лист хорошего backend-проекта
- [ ] README: эндпоинты, как запустить, переменные окружения, стек.
- [ ] Валидация входных данных на всех маршрутах.
- [ ] Централизованная обработка ошибок и корректные статус-коды.
- [ ] Секреты в `.env`, а не в коде (и `.env` в `.gitignore`).
- [ ] Пароли хешированы.
- [ ] Хотя бы несколько тестов на ключевые эндпоинты.
- [ ] Задеплоен (Railway/Render/Fly) или есть Docker-compose для запуска.
